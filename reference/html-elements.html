1. Document Structure The skeleton of every HTML page

Minimal HTML Page

Every HTML page starts with this structure. The browser reads it top to bottom.

<!DOCTYPE html>              Tells the browser: "This is HTML5"
<html lang="en">             Root element — language is English
  <head>                     Metadata (NOT visible on the page)
    ...
  </head>
  <body>                     Visible content goes here
    ...
  </body>
</html>

<!DOCTYPE html>

Not a tag — it's a declaration. Tells the browser to use modern HTML5 standards. Always the first line.

<html lang="en">

The root element that wraps everything. The lang attribute tells screen readers and search engines what language the page is in.

<head> vs <body>

Two required children of <html>:

<head>   → Metadata: title, CSS links, meta tags (NOT visible)
<body>   → Content: everything the user sees and interacts with
A common beginner mistake is putting <link> or <title> inside <body>. They belong in <head>.

2. Head Elements Metadata — not visible on the page

<meta charset="UTF-8">

Sets the character encoding. UTF-8 supports all languages and special characters.

<meta charset="UTF-8" />

<meta name="viewport">

Makes the page responsive on mobile devices. Without it, phones show a zoomed-out desktop view.

<meta name="viewport" content="width=device-width, initial-scale=1.0" />
width=device-width — page width matches the device screen.
initial-scale=1.0 — no zoom on load. Required for @media queries to work.

<title>

The text shown in the browser tab. Also used by search engines as the page title.

<title>Learning Log</title>
Where it appears
Learning Log ✕ +

<link>

Connects external resources — most commonly a CSS stylesheet.

<link rel="stylesheet" href="../static/css/main.css" />
rel="stylesheet" — the relationship is a stylesheet.
href — the path to the CSS file. From pages/main.html, go UP one level (../) then into static/css/.

3. Semantic Layout Elements Tags that describe their purpose

HTML5 introduced semantic elements that replace generic <div> tags. They make your code more readable and accessible.

<header>

The page header — typically contains logo, site name, and navigation or user info.

<header class="header">
  Logo, app name, username, logout button
</header>

<main>

The primary content area. There should only be ONE <main> per page. Content changes on each page while header/nav/footer stay the same.

<main class="content">
  Main Contents Here
</main>

<footer>

The page footer — typically contains copyright, credits, or additional links.

<footer class="footer">
  <h3>&copy; Learning Logs</h3>
</footer>

Semantic vs Generic Comparison

Old way (generic divs — works but meaningless):
<div class="header">...</div>
<div class="nav">...</div>
<div class="content">...</div>
<div class="footer">...</div>

Semantic way (tags describe their role):
<header>...</header>
<nav>...</nav>
<main>...</main>
<footer>...</footer>
Screen readers, search engines, and other developers can understand your page structure from the tags alone — without reading any CSS.

4. Text & Headings Displaying text content

Heading Tags <h1> through <h6>

Six levels of headings. <h1> is the largest, <h6> is the smallest. Use them in order — don't skip levels.

<h1>Heading 1</h1>   Main page title (use once per page)
<h2>Heading 2</h2>   Section title
<h3>Heading 3</h3>   Sub-section title (used in Learning Logs)
Heading sizes

h1 — Main Title

h2 — Section

h3 — Sub-section

h4 — Detail

h5 — Minor
h6 — Smallest

<p> — Paragraph

Block of text. Browsers add vertical spacing between paragraphs by default (removed by the * reset).

<p>Page Title</p>

5. Links & Images Navigation and media

<a> — Anchor (Link)

Creates a clickable link. The href attribute specifies where the link goes.

Link to another page:
<a href="topiclist.html">View Topics</a>

Link placeholder (goes nowhere yet):
<a href="#">Logout</a>

Styled as a button with a class:
<a href="#" class="logout">Logout</a>

Removing the underline with inline style:
<a href="#" style="text-decoration: none">No underline</a>
Link examples
Default link No underline Styled as button
href="#" is a placeholder — the link stays on the current page. In Week 4, these will point to actual servlet URLs.

<img> — Image

Displays an image. Self-closing tag (no </img>). Two required attributes:

<img src="../static/images/book.png" alt="LL" />

src — path to the image file

alt — text shown if the image fails to load, also read by screen readers

<img> is a self-closing tag — it ends with /> instead of having a closing tag. Other self-closing tags: <meta />, <link />, <input />, <br />.

7. Containers Grouping elements together

<div> — Block Container

A generic block-level container. Used to group elements for styling with CSS. Takes full width.

<div class="logo">
  <img src="book.png" alt="LL" />
  <h3>Learning Log</h3>
</div>
<div> has no semantic meaning — it's just a box. Use semantic tags (<header>, <nav>, etc.) when they fit. Use <div> only when no semantic tag applies.

<span> — Inline Container

An inline container for styling a piece of text within a block element. Does NOT start a new line.

<p>This is <span style="color: red;">important</span> text.</p>

This is important text.

8. Forms & Inputs Collecting user data

Forms let users send data to the server. Used in the workshop (add topic, add entry) and Week 4 (connected to Servlets).

<form>

Container for input fields. Two key attributes:

<form action="/addtopic" method="post">
  ...inputs go here...
</form>

action — the URL to send the data to (where the form submits)

method — how to send it: get (data in URL) or post (data in request body)

<form method="get"> vs <form method="post">

Browsers only support two form methods:

GET — data visible in URL (good for search)
<form action="/search" method="get">
  Result URL: /search?query=flexbox

POST — data hidden in request body (good for creating/updating)
<form action="/addtopic" method="post">
  Data sent in background, not visible in URL
Use GET for reading data (search, filter). Use POST for writing data (create, update, delete).

<label> + <input>

Labels describe input fields. The for attribute links a label to an input's id.

<label for="topicName">Topic Name:</label>
<input type="text" id="topicName" name="name" placeholder="Enter topic name" />
Live Example — click the label, cursor goes to the input

Input Types

The type attribute changes how the input looks and what data it accepts.

<input type="text" />       Single-line text
<input type="password" />   Hidden text (dots)
<input type="email" />      Validates email format
<input type="number" />     Numbers only
<input type="file" />       File upload
<input type="hidden" />     Not visible (sends data silently)
<input type="submit" />     Submit button
Input types

<textarea>

Multi-line text input. Unlike <input>, this has a closing tag.

<textarea name="description" rows="5" placeholder="Write your notes..."></textarea>
Textarea — resizable

<button>

A clickable button. More flexible than <input type="submit"> — can contain HTML inside.

<button type="submit">Save Topic</button>
<button type="button">Cancel</button>

Client-side Validation Attributes

HTML provides built-in validation — no JavaScript needed.

<input type="text" required />           Cannot be empty
<input type="email" required />          Must be valid email format
<input type="number" min="1" max="100" />  Range constraint
Try submitting with an empty required field
The required attribute and type validation happen in the browser BEFORE the form is sent. The server should still validate too.

Complete Form Example

A form for adding a topic — similar to what you'll build in the workshop.

<form action="/addtopic" method="post">
  <label for="name">Topic Name:</label>
  <input type="text" id="name" name="name"
         placeholder="Enter topic" required />
  <button type="submit">Save</button>
</form>

9. Common Attributes Properties that go on tags

Attribute Reference Table

Attributes are added inside the opening tag: <tag attribute="value">

Attribute Used On Purpose
class Any element CSS selector target — can be reused on multiple elements
id Any element Unique identifier — only ONE per page
href <a>, <link> URL or path to link to
src <img>, <script> Path to the resource file
alt <img> Alternative text if image fails or for screen readers
style Any element Inline CSS (prefer external stylesheets)
type <input>, <button> Input type (text, password, email, submit, etc.)
name <input>, <textarea> Key sent to server (e.g., name="topic" sends topic=value)
value <input>, <button> Current/default value of the field
placeholder <input>, <textarea> Ghost text shown when field is empty
required <input>, <textarea> Form won't submit if this field is empty
for <label> Links label to input by matching the input's id
action <form> URL where form data is sent on submit
method <form> HTTP method: get or post
rel <link> Relationship type (stylesheet)
lang <html> Page language (en for English)
charset <meta> Character encoding (UTF-8)
rows <textarea> Number of visible text lines

10. HTML Entities Special characters

Common Entities

Some characters have special meaning in HTML. Use entities to display them as text.

Entity Renders As When to Use
&copy; © Copyright symbol — used in footer
&lt; < Less-than — when you need literal < in text
&gt; > Greater-than — when you need literal > in text
&amp; & Ampersand — when you need literal & in text
&nbsp; (space) Non-breaking space — prevents line break
&quot; " Quote mark inside an attribute value
